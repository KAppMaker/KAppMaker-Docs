# First App and Configuration

In 5 minutes you will have both Android and iOS app. Once you have set up your environment according to the requirements, you can proceed to create your first app using KAppMaker. Follow the steps below for initial configuration:
You can also watch the video for configuration (5 minutes): [https://www.youtube.com/watch?v=5UdT3G2Kmx0](https://www.youtube.com/watch?v=5UdT3G2Kmx0).

### 1. Fork or Clone the Repository

Start by cloning or forking the KAppMaker repository to your local machine (ensure that the new repo has private access). KAppMaker is now distributed as a single monorepo containing the **KMP mobile app**, the **web (AI backend + landing page)**, and the **documentation**:

SSH:
```
git clone git@github.com:KAppMaker/KAppMaker-All.git
```
or HTTPS:
```
git clone https://github.com/KAppMaker/KAppMaker-All.git
```

The repo layout:
- `MobileApp/` — Kotlin Compose Multiplatform app (Android, iOS, JVM Desktop, Web/WASM)
- `Web/` — Firebase Hosting landing page + Cloud Functions (AI backend)
- `Documentation/` — this site (Docusaurus, git submodule)

### 2. Open In Android Studio

After cloning the repository, open the `MobileApp/` folder in Android Studio. On the first attempt, the project may fail to open due to missing keys related to authentication and subscription. In the `local.properties` file, add the following keys with dummy values:

```
GOOGLE_WEB_CLIENT_ID=test
SUBSCRIPTION_PROVIDER_ANDROID_API_KEY=test
SUBSCRIPTION_PROVIDER_IOS_API_KEY=test
```

### 3. Run Android App

After adding the above keys, sync the project again. Once the sync is complete, you should be able to run the Android app. 

### 4. Set Up the iOS App

If you successfully launch the Android app, proceed to set up the iOS app. The iOS app uses **Swift Package Manager** for its native dependencies (Firebase, Google Mobile Ads, etc.) — there is no CocoaPods step. Xcode resolves the packages automatically the first time you open the project.

### 5. Open the iOS Project in Xcode

Open `iosApp/iosApp.xcodeproj` in Xcode (there is no `.xcworkspace` — the project is SwiftPM-only). Wait for package resolution to finish, then run the app. At this point, you should be able to run both the Android and iOS apps.

### 6. Congratulations!

Congratulations! In just 5 minutes, you have set up both the Android and iOS apps. Your application already has authentication, notifications, and in-app purchase features configured (almost).


### 7. Change Application/Bundle Id and Name
Once you have successfully set up both the Android and iOS applications, you may want to change the application name and bundle ID to reflect your project. 
Run the `refactor_package.sh` script (from `MobileApp/`) to update your app ID and display name. Kotlin/Java package structure (`package com.example.app`) is independent of the `applicationId` / bundle ID :


```bash
./scripts/refactor_package.sh com.example.newapp NewApp
```

**Arguments:**
- `<newAppId>` (required, 1st positional) → Android `applicationId` / iOS bundle ID  
- `<newAppName>` (required, 2nd positional) → App display name  
- `--skip-package-rename` (optional) → Keep Kotlin packages and directories unchanged; only update the `applicationId`, bundle ID, Firebase refs, and app name. Makes it easier to sync changes across multiple apps from the same codebase and avoid merge conflicts in future.

:::tip Important

It is advisable to create a backup of your project (or commit) before proceeding, in case any errors occur during the process. The script prints a plan and asks for confirmation before changing anything. Before running it, ensure that Xcode is closed. Open it again after the changes are completed to reflect the updates.

:::


Re-sync Gradle once and rebuild to ensure all changes are applied correctly.


### 8. Keeping Your Repository Up to Date

To keep your repository updated with the latest changes,
first set your own github repository as origin, then merge changes from the KAppMaker monorepo:

```
git remote set-url origin <new-url> (YOUR GITHUB REPO)
git remote add upstream git@github.com:KAppMaker/KAppMaker-All.git
git fetch upstream
git merge upstream/main
```
